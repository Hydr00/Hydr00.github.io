The working process of crackers

1. Collect information in the network system

The collection of information does not harm the target, but only provides useful information for further intrusion. Crackers may use the following public protocols or tools to collect information about individual host systems that reside in a network system

2. Detecting the security vulnerabilities of the target network system

After gathering some information ready to attack the target, Crackers detects each host on the target network to look for security vulnerabilities within the system

3. Establish a simulated environment and carry out simulated attacks.

According to the information obtained from the first two small points, a simulation environment similar to the attack object is established, and then a series of attacks are carried out on the simulated target. During this period, by checking the logs of the attacked party and observing the response of the detection tool to the attack, we can further understand the "marks" left in the attack process and the status of the attacked party, so as to develop a more detailed attack strategy.

4. Specific implementation of network attacks

According to the information obtained in the previous steps, the intruder summarizes the corresponding attack methods according to his own level and experience. After the practice of simulation attack, the intruder will wait for the opportunity to implement the real network attack.
